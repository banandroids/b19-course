I.013-0 Задание от Кинопоиска II

Задание: Кинопоиск пишет базу данных на Котлине для хранения фильмов и просит вас создать метод search для поиска фильмов по названию и рейтингу.

Кинопоиск уже создал класс фильма Film с полями:

  • name - название фильма (строка)
  • rating - рейтинг фильма (дробное число, от 0 до 10)

Нужно написать метод search с параметрами:

  • films - полный список с фильмами
  • x - текст, который ищет в названии фильма пользователь
  • y - минимальный рейтинг фильма
Этот метод должен вернуть только те фильма, в названии которых есть x (любыми буквами - большими или маленькими) и рейтинг которых больше или равен y.

Пример: Для тестовых данных из main() выражение search(testFilms, "ёлки", 6f) должно вернуть список из одно фильма "Ёлки" (потому что "Ёлки 5" не проходят по рейтингу).

Теория:

Списки (или листы, потому что List) - типа массива, только удобнее: можно изменить размер после создания (а у массивов нельзя) и больше полезных методов. Минусы списков: массивы быстрее работают и занимают меньше памяти.

Создание списков:
// неизменяемые списки (List) - после создания ни число элементов,
// ни сами элементы менять в них нельзя
val list: List<String> = listOf("привет", "пока") // создание неизменяемого списка из двух элементов
val list = listOf<String>("привет") // можно тип элемента писать после listOf
val list = listOf("привет") // а можно совсем не указывать тип, он из
